<objective>
Fix BUG-03: provision.sh line 47 hardcodes the Coolify server name lookup as `coolify_get_server_uuid "localhost"`. Coolify server names are user-configurable; not every install uses "localhost". On a Coolify instance where the node has a custom name, provision fails with `ERROR: server 'localhost' not found`.

Purpose: Make the server name configurable per machine via `~/.claude/coolify.json` while preserving the existing default ("localhost") so single-node installs keep working without any config change.

Output:
1. provision.sh reads optional `server_name` from `coolify.json` (default `"localhost"`) using the same pattern as `ssh_host` and passes it to `coolify_get_server_uuid`.
2. docs/schema.md documents the new optional field, including the backward-compatibility note (the existing `ssh_host` migration block is the model).
</objective>

2. server_name is OPTIONAL — do NOT fail if missing; just use the default
